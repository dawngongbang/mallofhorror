# 몰오브호러 온라인 (Mall of Horror Online)
## 프로젝트 기획서 v2.3

> 작성일: 2026-03-23
> 최종 수정: 2026-03-26 (아이템 효과 확정, 좀비 플레이어, weapon_use 페이즈, 공지 시스템 추가)
> 목적: 몰오브호러 보드게임을 서버 없이 친구들과 온라인으로 플레이하기 위한 웹 앱 구현

---

## 1. 게임 개요

몰오브호러는 3~6인 협상/배신 보드게임이다.
쇼핑몰을 배경으로 각 플레이어가 동일한 캐릭터 세트를 조종하며,
매 라운드 좀비의 습격을 피해 살아남고 최고 점수를 획득하는 것이 목표다.

---

## 2. 맵 구성

주차장(4번)이 중앙에 있고 나머지 5개 구역이 오각형 형태로 배치.
**이동 시 배치는 관계없음 — 모든 구역 간 자유 이동.**

```
           [5. 보안실]
          (수용3/방어3)

[6. 슈퍼마켓]         [1. 화장실]
 (수용6/방어4)         (수용3/방어3)

        [4. 주차장]
       (수용∞/방어0)

[3. 장난감가게]       [2. 옷가게]
  (수용4/방어4)        (수용4/방어4)
```

### 구역별 특성

| 번호 | 구역 | 최대 수용 | 방어 한도 | 특수 이벤트 |
|------|------|----------|----------|-----------|
| 1 | 화장실 | 3명 | 3 | 없음 |
| 2 | 옷가게 | 4명 | 4 | 없음 |
| 3 | 장난감가게 | 4명 | 4 | 없음 |
| 4 | 주차장 | 무제한 | 0 | 트럭 수색 투표 (덱에 아이템 있을 때만) |
| 5 | 보안실 | 3명 | 3 | 보안관 투표 |
| 6 | 슈퍼마켓 | 6명 | 4 | 없음 |

> **방어 한도**: 캐릭터가 아무리 많아도 이 수치를 초과한 방어는 불가.
> **주차장**: 방어 한도 0 → 좀비가 1마리라도 있으면 항상 습격 발생.

---

## 3. 캐릭터

각 플레이어는 동일한 캐릭터 세트를 소유하고 시작한다.

| 캐릭터 | 점수 | 방어력 | 투표권 | 특성 |
|--------|------|--------|--------|------|
| 미녀(Belle) | 7점 | 1 | 1 | 미녀 집결 구역 판정 기준 |
| 터프가이(Tough Guy) | 5점 | **2** | 1 | 좀비 2마리 방어 |
| 건맨(Gunman) | 3점 | 1 | **2** | 투표권 2개 |
| 아이(Kid) | 1점 | 1 | 1 | — |

**플레이어 수별 캐릭터 세트:**
- 3인: 미녀 + 터프가이 + 건맨 + 아이 (인당 4캐릭터)
- 4~6인: 미녀 + 터프가이 + 건맨 (인당 3캐릭터)

### 방어력 계산
```
구역 방어력 = min(구역 방어 한도, 구역 내 캐릭터 방어력 합산 + 아이템 보너스)

예) 슈퍼마켓에 터프가이(2) + 미녀(1) + 미녀(1) + 건맨(1) = 합산 5
    min(5, 4) = 4 → 좀비 5마리면 뚫림
```

## 4. 아이템

게임 시작 시 전체 21장의 아이템 덱에서 각 플레이어에게 3장씩 랜덤 지급.

| 아이템 | 장수 | 사용 타이밍 | 효과 |
|--------|------|------------|------|
| 숨기(Hidden Card) | 3 | weapon_use 페이즈 | 내 캐릭터 1명을 이번 구역 처리가 끝날 때까지 숨김. 숨은 캐릭터는 방어·투표 참여·투표 대상에서 제외. 구역 처리 완료 후 자동 등장 |
| 보안카메라(CCTV) | 3 | 아무때나 | 이번 라운드 주사위 결과를 보안관과 동일하게 확인 가능 |
| 스프린트(Sprint) | 3 | 미구현 | 미정 |
| 협박(Threat) | 3 | 투표 중 | 내 투표권 +1 (1회성, 즉시 적용) |
| 하드웨어(Hardware) | 3 | 미구현 | 습격 판정 시 방어력 +1 |
| 도끼 | 1 | weapon_use 페이즈 | 좀비 1마리 제거 |
| 권총 | 1 | weapon_use 페이즈 | 좀비 1마리 제거 |
| 샷건 | 1 | weapon_use 페이즈 | 좀비 2마리 제거 |
| 야구배트 | 1 | weapon_use 페이즈 | 좀비 1마리 제거 |
| 수류탄 | 1 | weapon_use 페이즈 | 좀비 2마리 제거 |
| 전기톱 | 1 | weapon_use 페이즈 | 좀비 2마리 제거 |

**총 21장** (일반 5종×3 + 무기 6종×1)

### weapon_use 페이즈 동작 방식

```
좀비 습격 판정 직전 15초 동안:
  - 해당 구역 플레이어들이 아이템 패널에서 무기·숨기 아이템 선택
  - 무기: 여러 장 선택 가능, kill 수 합산
  - 숨기: 1장만, 내 캐릭터 1명 선택 (구역 내 내 캐릭터가 1명이면 자동 선택)
  - 선택 후 "완료" 버튼으로 확정 → 모두 확정되면 즉시 조기 종료
  - 타임아웃 시 미확정자는 패스 처리

전원 확정 후:
  1. 무기 kill 일괄 적용 (선택한 무기 인벤토리에서 소모)
  2. 숨기 아이템 효과 적용 (캐릭터 숨김 상태 진입)
  3. 공지:
     - 무기 사용 시: "○○님이 N마리 처치 → 남은 좀비 M마리"
     - 숨기 사용 시: "○○님의 ×× 캐릭터가 사라졌습니다"
  4. 업데이트된 좀비 수로 습격 재판정
```

### 숨기 아이템 타이밍 상세

```
weapon_use 페이즈: 숨기 선택 → pendingHideChoices에 임시 저장 (비공개)
전원 확정 후: 호스트가 hiddenCharacters에 이동 (공개) + "사라졌습니다" 공지
이후 투표 포함 해당 구역 처리 진행:
  - 숨은 캐릭터는 방어/투표에서 제외
  - 시각적으로 반투명 처리 (🫥 아이콘)
구역 처리 완료(zone_announce 종료) 시:
  - "나타났습니다" 공지 → 1.5초 후 다음 구역으로 전환
  - hiddenCharacters 초기화
```

## 5. 게임 흐름

### 5.1 초기 세팅

```
1. 각 플레이어에게 아이템 3장 랜덤 지급
2. 보안관 랜덤 선정 (임시 보안관)
3. 플레이어 순서 확정 (게임 내내 유지)
4. 캐릭터 배치 페이즈:
   - 보안관부터 순서대로 주사위 2개 굴림
   - 나온 두 구역 번호 중 하나 선택 → 자기 캐릭터 1개 배치
   - 해당 구역이 가득 찼으면 원하는 곳에 자유 배치
   - 모든 플레이어가 캐릭터를 다 배치할 때까지 반복
5. 세팅 완료
```

### 5.2 첫 라운드 시작

```
첫 라운드는 임시 보안관만 있음 (보안실 점유 시 진짜 보안관)
→ 주사위 4개 굴리기만 (좀비 배치) → 이동 없이 바로 이벤트 페이즈
```

### 5.3 일반 라운드

```
[1단계] 주사위 굴리기
  - 보안관이 주사위 굴리기 요청 버튼 클릭 → 호스트가 처리
  - 진짜 보안관(보안실 점유): 결과 혼자 확인
    → 공개 / 거짓말 / 침묵 자유
  - 보안카메라 아이템 사용자도 확인 가능
  - 임시 보안관: 전체 공개
  - 좀비 플레이어(전원 사망): 이 단계에서 좀비 배치할 구역 선택

[2단계] 이동 페이즈
  ① 보안관부터 순서대로 이동할 캐릭터 공개 선언 (본인 소유만)
  ② 모든 플레이어가 목적지 구역 번호 비공개 선택
  ③ 동시 공개
  ④ 보안관 순서대로 이동 처리
     - 같은 구역으로는 이동 불가 (반드시 다른 구역)
     - 목적지가 가득 찼으면 → 주차장으로 튕겨남

[3단계] 좀비 배치 + 이동 완료 공개
  A. 주사위 결과 기반 구역에 좀비 배치
  B. 추가 좀비 배치:
     - 미녀가 가장 많은 구역 +1 (동률이면 스킵)
     - 캐릭터가 가장 많은 구역 +1 (동률이면 스킵)
     - 좀비 플레이어가 선택한 구역 +1 (공지: "좀비가 된 ○○님이 ~에 나타났습니다!")

[4단계] 이벤트 페이즈 (1번 구역부터 6번까지 순서대로)
  각 구역 처리 순서:
  1. event 페이즈: 구역 진입 (zone_announce로 자동 전환)
  2. zone_announce 페이즈:
     - 좀비 수 > 방어력? → weapon_use 페이즈 → 무기/숨기 사용 (15초)
     - 무기 적용 후 재판정 → 습격이면 voting 페이즈
     - 주차장/보안실: 추가 투표 이벤트
     - 4초 후 다음 구역으로 전환 (숨은 캐릭터 있으면 reveal 공지 후 전환)
  3. 6개 구역 처리 완료 → 라운드 종료

[5단계] 승리 조건 체크
```

### 5.8 좀비 플레이어

```
모든 캐릭터가 죽어 생존자가 없는 플레이어는 "좀비 플레이어"가 된다.
매 라운드 roll_dice / dice_reveal 페이즈 중 배치할 구역을 선택.
이동 완료 후 일반 좀비와 동시에 선택 구역에 +1 좀비 배치.
공지: "좀비가 된 ○○님이 [구역명]에 나타났습니다!"
```

### 5.4 습격 처리

```
습격 발생 시:
  1. 해당 구역 플레이어들끼리 투표
     (투표권: 캐릭터당 1표, 건맨은 2표, 협박카드 사용 시 +1표)
  2. 가장 많이 지목된 플레이어의 캐릭터 사망
     - 해당 구역에 캐릭터가 1개면 그 캐릭터 사망
     - 여러 개면 플레이어가 선택
  3. 사망 발생 → 해당 구역 좀비 전부 소멸

동률 처리:
  1차 동률 → 재투표
  2차 동률 → 해당 구역 밖 전체 플레이어도 참여
  이후 동률 → 동률 깨질 때까지 계속 투표

주차장 특수:
  - normal 모드: 위와 동일 (투표로 1명 사망)
  - hardcore 모드: 좀비 수만큼 사망자 발생 (투표로 순서 결정)
```

### 5.5 특수 구역 이벤트

```
4번 주차장 (해당 구역에 캐릭터 있을 때):
  습격 처리 완료 후 → 트럭 수색자 투표 (덱에 아이템이 1장 이상 있을 때만)
  → 당선자: 남은 아이템 덱에서 min(3, 남은 수)장을 랜덤으로 뽑아 본인만 확인
  → 당선자가 각 카드를 어떻게 처분할지 선택:

  덱 잔량별 선택지:
  - 3장 이상: 랜덤 3장 공개 → 1장 소유 선택 / 1장 증정 대상 선택 / 나머지 1장 자동 반납
  - 2장:      랜덤 2장 공개 → 1장 소유 선택 / 나머지 1장 증정 대상 선택
  - 1장:      1장 자동 획득 (선택 없음)
  - 0장:      트럭 수색 스킵 (투표 자체 미발생)

5번 보안실 (해당 구역에 캐릭터 있을 때):
  습격 처리 완료 후 → 보안관 투표
  → 동률 2회까지: 기존 보안관 유지
  → 당선자: 다음 라운드부터 진짜 보안관
  → 보안관은 주사위 결과를 혼자 확인 + 공개 여부 자유
```

### 5.6 보안관 시스템

```
임시 보안관: 랜덤 선정, 주사위 결과 공개
진짜 보안관: 보안실 투표로 선출, 보안실에 캐릭터가 있어야 유지
  - 보안실 캐릭터 없으면 → 다시 임시 보안관 권한만

보안관 교체 시: 다음 라운드부터 턴 순서 재편
  (기존 플레이어 상대 순서 유지, 새 보안관이 1번)
```

### 5.7 승리 조건

```
매 라운드 종료 시 체크:
  전체 생존 캐릭터 수 ≤ 4
  AND 주차장을 제외한 한 구역에 모든 생존 캐릭터가 집결
  → 게임 종료

생존 캐릭터 점수 합산:
  미녀 7점 / 터프가이 5점 / 건맨 3점 / 아이 1점
  → 합산 점수 1등 플레이어 승리 (동점 시 공동 우승)
```

## 6. 공지 시스템

게임 중 주요 이벤트는 화면 하단에 고정 오버레이 배너로 표시된다.

| 공지 종류 | 트리거 | 표시 내용 | 자동 해제 |
|-----------|--------|----------|----------|
| 숨기 공지 (hide) | weapon_use 종료 시 숨기 확정 | "○○님의 ××가 ~~에서 사라졌습니다" | 5초 후 |
| 등장 공지 (reveal) | zone_announce 종료 시 | "사라졌던 ○○님의 ××가 ~~에서 모습을 드러냈습니다" | 5초 후 |
| 무기 사용 결과 | weapon_use 종료 후 (kill > 0일 때만) | "○○님이 N마리 처치, 총 M마리 처치 → 남은 좀비 K마리" | 5초 후 |
| 좀비 플레이어 | 이동 완료 후 | "좀비가 된 ○○님이 [구역]에 나타났습니다!" | 5초 후 |
| 투표 결과 | 전원 투표 확정 후 4초 | 득표수, 패배자 등 | 투표 결과 처리 후 |
| 좀비 공격 결과 | 투표 해결 후 | 패배자 및 사망 캐릭터 | zone_announce 이후 |
